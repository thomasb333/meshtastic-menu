Meshtastic Menu Tree

Meshtastic App Top-Level (Android)
  • Connection
    • Bluetooth Pairing/Selection
      Pair and select Bluetooth devices for connection to Meshtastic hardware. Default: N/A
    • WiFi/IP Connection
      Connect to Meshtastic devices via WiFi or Ethernet IP address. Default: N/A
    • Serial/USB Connection
      Connect via serial USB for direct hardware access, useful for debugging or CLI. Default: N/A
  • Messages
    • Threaded Conversations
      View and participate in threaded message conversations across the mesh. Default: N/A
    • Quick Chat Actions (e.g., thumbs up, location share)
      Quick actions like reactions (thumbs up) or sharing location in chats. Default: N/A
  • Map
    • Node Locations
      Display locations of nodes on the map based on shared positions. Default: N/A
    • Waypoints
      Create and view waypoints for navigation or points of interest. Default: N/A
    • Offline Maps Download
      Download maps for offline use in areas without internet. Default: N/A
  • Nodes
    • Node List (online/offline, SNR, hops)
      List of nodes with status (online/offline), signal-to-noise ratio (SNR), and hop count. Default: N/A
    • Remote Admin (via Admin Channel)
      Administer remote nodes via the Admin channel for configuration changes. Default: N/A
  • Settings
    • App Settings
      • Theme (Light, Dark, System)
        Choose app theme: Light, Dark, or follow System settings. Default: System
      • Map Settings (Online/Offline, Tile Sources)
        Configure map type (online/offline) and tile sources for custom maps. Default: Online
      • Message Notification Settings
        Customize notifications for incoming messages. Default: Enable
      • Debug Logs
        View and export debug logs for troubleshooting. Default: N/A
      • Device DB Cache Limit
        Set limit for device database cache to manage storage. Default: Unlimited
      • System Settings (Permissions)
        Manage system permissions like location, Bluetooth, etc. Default: N/A
    • Radio Configuration
      • User
        • Long Name
          A personalized name for your device. If you are a licensed HAM operator and have enabled IsLicensed, this should be set to your HAM operator call sign. Default: Generated
        • Short Name
          A personalized short identifier for your device. Default: 4 chars
        • Team
          Team color for ATAK integration, to categorize nodes in teams. Default: None
        • Is Licensed Operator
          If you are a licensed HAM operator and have considered the privileges and restrictions of using Meshtastic with a HAM license, enable this flag. Default: Disable
        • MAC Address
          MAC Address of the device, read-only. Default: N/A
      • Channels (Up to 8 channels)
        • Channel List
          • For each channel:
            • Name
              A short identifier for the channel. (< 12 bytes). Default: Empty
            • Role
              Role of the channel: DISABLED, PRIMARY, SECONDARY. Default: PRIMARY for first channel
            • PSK
              The encryption key used for private channels. Hex byte 0x01 for the Primary default channel. Must be either 0 bytes (no crypto), 16 bytes (AES128), or 32 bytes (AES256). Default: Random
            • Uplink Enabled
              If enabled, messages from the mesh will be sent to the public internet through any node's configured gateway. Default: Disable
            • Downlink Enabled
              If enabled, messages captured from a public internet gateway will be forwarded to the local mesh. Default: Disable
            • Modem Preset
              Inherits from LoRa or custom modem preset. Default: Inherit
            • Module Permissions
              Bitfield to allow specific modules like Telemetry. Default: All allowed
        • QR Code/Share URL for Channel
          QR Code or shareable URL for the channel. Default: N/A
      • Device
        • Role
          Sets the role of the device (e.g., CLIENT, ROUTER, etc.). See the table for descriptions of each role. Default: CLIENT
        • Rebroadcast Mode
          Defines the device's behavior for how messages are rebroadcasted. Default: ALL
        • Button GPIO
          This is the GPIO pin number that will be used for the user button, if your device does not come with a predefined user button. Default: Hardware-specific
        • Buzzer GPIO
          This is the GPIO pin number that will be used for the PWM buzzer, if your device does not come with a predefined buzzer. Default: Hardware-specific
        • Buzzer Mode
          This is the mode for the buzzer. Default: ALL_ENABLED
        • Node Info Broadcast Secs
          This is the number of seconds between NodeInfo message (containing i.a. long and short name) broadcasts from the device. Default: 900
        • Double Tap as Button Press
          This option will enable a double tap, when a supported accelerometer is attached to the device, to be treated as a button press. Default: Disable
        • Disable Triple Click
          Disable triple click (for GPS). Default: Disable
        • TZDEF
          The tzdef setting allows the local offset to be defined for the device. It uses the TZ Database format to display the correct local time on the device display and in its logs. Default: UTC
        • LED Heartbeat Disabled
          Disable LED heartbeat. Default: Disable
      • Position
        • Position Broadcast Secs
          If smart broadcast is off we should send our position this often. Default: 900
        • Smart Position Enabled
          Smart broadcast will send out your position at an increased frequency only if your location has changed enough for a position update to be useful. Default: Enable
        • Fixed Position
          If set, this node is at a fixed position. Default: Disable
        • GPS Mode
          Configures whether the GPS functionality is enabled, disabled, or not present on the node. Default: ENABLED if present
        • GPS Update Interval
          How often we should try to get GPS position (in seconds). Default: 30
        • Position Flags
          Defines which options are sent in POSITION messages. Default: ALTITUDE | DOP | HEADING | SPEED
        • RX GPIO
          RX GPIO for GPS. Default: Hardware-specific
        • TX GPIO
          TX GPIO for GPS. Default: Hardware-specific
        • Broadcast Smart Min Distance
          The minimum distance in meters traveled (since the last send) before we can send a position to the mesh if smart broadcast is enabled. Default: 30
        • Broadcast Smart Min Interval Secs
          The minimum number of seconds (since the last send) before we can send a position to the mesh if smart broadcast is enabled. Default: 120
        • GPS EN GPIO
          GPS EN GPIO. Default: Hardware-specific
      • Power
        • Is Power Saving
          When activated, this feature disables Bluetooth, Serial, WiFi, and the device's screen to conserve power. Default: Disable
        • On Battery Shutdown After Secs
          Automatically shut down a device after a defined time period if power is lost. Default: 0
        • ADC Multiplier Override
          Ratio of voltage divider for battery pin e.g. 3.20. Default: Hardware-specific
        • Wait Bluetooth Secs
          How long to wait before turning off BLE in no Bluetooth states. Default: 60
        • SDS Secs
          Super deep sleep interval. Default: 0
        • LS Secs
          Light sleep interval. Default: 300
        • Min Wake Secs
          Minimum time till sleep. Default: 10
        • Device Battery INA Address
          If an INA-2XX device is auto-detected on one of the I2C buses at the specified address, it will be used as the authoritative source for reading device battery level voltage. Default: 0x40
        • Powermon Enables
          Power monitoring enables. Default: 0
      • Network
        • WiFi Enabled
          Enables WiFi on the device. Default: Disable
        • WiFi SSID
          SSID for the WiFi network. Default: Empty
        • WiFi PSK
          PSK for the WiFi network. Default: Empty
        • NTP Server
          NTP server to use for time synchronization. Default: meshtastic.pool.ntp.org
        • ETH Enabled
          Enables Ethernet if supported. Default: Disable
        • Address Mode
          DHCP or STATIC. Default: DHCP
        • IPv4 Config
          Sub: IP, Gateway, Subnet, DNS. Default: DHCP
        • Rsyslog Server
          Rsyslog server. Default: Empty
        • Enabled Protocols
          Bitfield, e.g., UDP_BROADCAST. Default: 0
        • IPv6 Enabled
          Enable IPv6. Default: Disable
      • Display
        • Screen On Secs
          How long the screen should remain on after user interaction. Default: 60
        • Auto Screen Carousel Secs
          Interval for automatic screen carousel. Default: 5
        • Flip Screen
          Flip screen 180 degrees. Default: Disable
        • Units
          METRIC or IMPERIAL. Default: METRIC
        • OLED Type
          AUTO, SSD1306, SH1106, etc. Default: AUTO
        • Display Mode
          DEFAULT, TWOCOLOR, INVERTED, COLOR. Default: DEFAULT
        • Heading Bold
          Make heading bold. Default: Enable
        • Wake On Tap Or Motion
          Wake on tap or motion. Default: Enable
        • Compass Orientation
          DEGREES_0, etc. Default: DEGREES_0
        • Use 12h Clock
          Use 12h clock. Default: Disable
        • Use Long Node Name
          Use long node name. Default: Disable
        • Enable Message Bubbles
          Enable message bubbles. Default: Enable
      • LoRa
        • Use Preset
          Use preset modem settings. Default: Enable
        • Modem Preset
          LONG_FAST, LONG_MODERATE, etc. Default: LONG_FAST
        • Region
          UNSET, US, EU_433, etc. Default: UNSET
        • Bandwidth
          Bandwidth. Default: Preset-dependent
        • Spread Factor
          Spread Factor 7-12. Default: Preset-dependent
        • Coding Rate
          Coding Rate 5-8. Default: Preset-dependent
        • Frequency Offset
          Frequency offset. Default: 0
        • Hop Limit
          Hop limit. Default: 3
        • TX Enabled
          TX enabled. Default: Enable
        • TX Power
          TX power. Default: Max legal
        • Channel Num
          Channel num. Default: 0
        • Override Duty Cycle
          Override duty cycle. Default: Disable
        • SX126X RX Boosted Gain
          SX126X RX boosted gain. Default: Disable
        • Ignore Incoming
          Ignore incoming. Default: Empty
        • Override Frequency
          Override frequency. Default: 0
        • Ignore MQTT
          Ignore MQTT. Default: Disable
        • PA Fan Disabled
          PA fan disabled. Default: Disable
      • Bluetooth
        • Enabled
          Enabled. Default: Enable
        • Mode
          RANDOM_PIN, FIXED_PIN, NO_PIN. Default: RANDOM_PIN
        • Fixed Pin
          Fixed pin. Default: 123456
      • Security
        • Serial Enabled
          Serial enabled. Default: Enable
        • Debug Log Enabled
          Debug log enabled. Default: Disable
        • Public Key
          Public key. Default: Generated
        • Private Key
          Private key. Default: Generated
      • Module Settings
  • Debug
    • Logs
      View system logs for troubleshooting. Default: N/A
    • Factory Reset
      Reset device to factory settings. Default: N/A
    • Share Debug Info
      Share debug information for support. Default: N/A

Meshtastic Web Flasher (PC)
  • Select Device
    Choose the hardware device to flash (e.g., Heltec V3). Default: N/A
  • Software Version
    Select firmware version to flash (stable or beta). Default: Latest stable
  • Flash Firmware
    Flash the selected firmware to the device. Default: N/A
  • Initial Configuration
    • Region
      Set LoRa region for frequency compliance. Default: UNSET
    • Modem Preset
      Choose LoRa preset for range/speed. Default: LONG_FAST
    • User Name
      Set long and short names. Default: Generated
    • Channels
      Configure primary and secondary channels. Default: Public
    • Advanced Options
      Access full radio config via connected web client after flashing. Default: N/A
  • Radio Configuration (Post-Flash Web Client)
